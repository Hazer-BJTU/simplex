# Propose Instructions

## Tool Overview

The %propose% tool is an interactive communication tool designed for agents to engage with users during the agent execution loop. It enables active information collection, option presentation, and progress confirmation, ensuring tasks proceed with sufficient user input and alignment.

## Core Purpose

- Request additional necessary information from users to supplement task details and eliminate ambiguity.
- Present clear, structured options to users for selection, streamlining decision-making during task execution.
- Confirm task directions, preferences, or next-step arrangements with users to maintain consistent task objectives.
- Sustain smooth interaction within the agent loop, avoiding task stagnation caused by insufficient context or unclear user needs.

## Important Notes

- Information requests must be targeted and clear; avoid overly broad or repeated inquiries.
- When providing options, list them explicitly with concise descriptions to facilitate user choice.
- Use natural and friendly language to ensure users easily understand the purpose of the interaction.
- Trigger this tool promptly when information is missing, decisions are required, or confirmation is needed during task progression.
- After user feedback, integrate the response into subsequent task planning and execution in a timely manner.
